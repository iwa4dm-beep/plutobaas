# PITR drill — latest measured RPO/RTO + restore correctness

This page is the **durable artifact** of our cross-region point-in-time
recovery drill. It is regenerated by CI on every merge to `main` from
`backend/scripts/pitr-drill.sh`, uploaded as a workflow artifact
(`pitr-drill-report`, retention 90 days) and mirrored here so the last
known-good numbers survive artifact expiry.

- Runbook: [`docs/runbooks/pitr-drill.md`](../runbooks/pitr-drill.md)
- Live status: [`/status`](/status)
- Machine-readable report: attached as `pitr-drill-report.md` to the most
  recent `backend » cross-region PITR drill` workflow run.

## Targets (SLO)

| Metric | Target |
|---|---|
| RPO (data loss window) | ≤ 60s |
| RTO (time to first byte on replica) | ≤ 15 min |
| Restore correctness | dry-run WAL coverage passes to `target_time` |
